01 · The problem03
01 THE PROBLEM

Misinformation moves faster than the team can Google.

Slack is the substrate that modern teams use to trade information at speed. That speed is also why half-true claims spread before anyone has time to challenge them.

The person who eventually pushes back has to do their homework alone, in a side tab, then disrupt the thread to share it. By that point the original claim has been internalised by three other people and re-shared in a different channel.

What today looks like

WITHOUT VERITYPE

  • Claim drops in #general at 09:47.
  • Three emoji-reactions, no challenge.
  • 10:30, someone tries to push back. Has to Google for ten minutes.
  • Source debate derails the channel for the rest of the morning.
  • The team's track record on disputed claims is invisible.

WITH VERITYPE

  • Bot offers a one-tap verification, privately to the speaker.
  • Verdict arrives in 5 to 8 seconds with weighted citations.
  • Card stays in thread. Sources are one click away.
  • Channel stays calm, the work resumes.
  • Workspace dashboard tracks claims, verdicts, latency, coverage.

Why a fact-checking agent, not a fact-checking link

People do not stop to verify claims when the cost is opening another tab. People do stop to verify when the cost is a single button. Slack apps that are not in-channel get used twice and forgotten. Veritype lives in the conversation, with the same APP pill any other bot has, and never makes you context-switch.

Why a font for the chart? A PNG chart is cute. A chart that is also valid text inside a Datatype-styled card is searchable, copy-pasteable, scales with text size, and gives the verdict an inarguably unique visual identity. Nobody else in this hackathon is doing it.

02 WHAT IT DOES

Listen, verify, render.

The pipeline

Step What happens Model
Listen Every channel message hits a triage classifier that decides whether it asserts a verifiable, public-interest claim. If yes, an ephemeral "want me to verify this?" prompt appears. claude-haiku-4-5
Verify On accept, a research agent fans out web searches, weighs sources by quality, and synthesises a verdict (TRUE / MOSTLY TRUE / MIXED / MISLEADING / FALSE / UNVERIFIED) with confidence, citations, and context. claude-opus-4-7 + web_search
Render The verdict is painted into a Datatype-font HTML card and rasterised by Puppeteer. Slack thread gets a PNG plus a Block Kit mrkdwn fallback for accessibility, search, and mobile. Puppeteer + Datatype

Three Datatype expressions, three insights

The font turns text like {b:20,45,70} into a real bar chart through ligature substitution. Every verdict card uses three of these:

  • {b:…} source-weight bars: primary documents sit taller than blog posts.
  • {p:…} three pies for supports / refutes / context as evidence balance.
  • {l:…} a sparkline of how often the claim has been showing up over time.

03 · Architecture05
03 ARCHITECTURE

One brain, three surfaces.

Everything lives in src/. The Slack bot, the MCP server, and the demo dashboard are three thin shells around a single core module.

┌───────────────┐ ┌────────────────┐ ┌─────────────────┐ │ Slack Bolt │ │ MCP server │ │ CLI / Vite UI │ │ (Socket) │ │ (stdio) │ │ (Playground) │ └───────┬───────┘ └────────┬───────┘ └────────┬────────┘ │ │ │ └─────────┬───────────┴─────────┬───────────┘ ▼ ▼ ┌───────────────────────────────────┐ │ veritype/core (TypeScript) │ │ screen → research → render │ └────────────────┬──────────────────┘ ▼ ┌─────────────────────────────────┐ │ Anthropic Haiku 4.5 (screen) │ │ Anthropic Opus 4.7 + web_search │ │ Puppeteer + Datatype font (png) │ └─────────────────────────────────┘

Why split the brain from the bot

If the fact-checking logic only lived inside the Slack handler, a teammate using Claude Desktop could not call it. By isolating it in src/core and re-exposing it through MCP, the same verdict that lands in a Slack thread can be requested from any MCP-aware client.

Hackathon requirement check. Slack Agent Builder Challenge requires at least one of: Slack AI capabilities, MCP server integration, Real-Time Search API. Veritype hits two: a real MCP server and Anthropic's web_search_20250305 tool used by every verification.

Failure modes built in

  • If Opus is unavailable or rate-limited, the researcher falls back to claude-sonnet-4-6 with the same JSON contract.
  • If the screener returns malformed JSON, the message is silently skipped (no offer, no error pollution).
  • If Puppeteer fails to render, the bot still posts the mrkdwn Block Kit fallback so users always see the verdict.

05 · In-channel UX07
05 IN-CHANNEL UX

The proactive offer.

Veritype's most distinctive UX move: it asks first. Every message in a watched channel goes through the Haiku screener, and when a verifiable claim is detected, the bot drops an ephemeral prompt visible only to the speaker. No public noise, no shaming.

Three properties that make it feel native

  1. Ephemeral by default. The offer is only seen by the message author. If they dismiss, nobody else ever knew.
  2. Threaded by default. Verdicts post in thread so the main channel stays scannable.
  3. Block Kit + image dual surface. The card is sent both as a Datatype PNG and as native Block Kit mrkdwn, so it works on mobile, screen readers, and Slack search.

06 · The verdict card08
06 THE VERDICT CARD

Anatomy of a Datatype card.

Every verdict card is the same template, rendered with the verdict-specific accent color. The three Datatype expressions across the metrics row tell you the whole story without reading a single sentence.

Reading the card

ElementTells you
Verdict pillTRUE / MOSTLY TRUE / MIXED / MISLEADING / FALSE / UNVERIFIED with a confidence pie and percentage.
Claim lineThe standardised one-sentence rephrasing the agent verified, not the raw message.
One-linerThe bottom line in 140 characters, no hedging.
Sources weightedBar chart of every source's quality score. Tall bars are primary documents, short bars are blog re-treads.
Evidence balanceThree pie charts for supports / refutes / context. Tells you whether the agent saw mostly corroboration or a real fight.
Salience over timeSparkline showing how often the claim has been appearing across the public record recently. Spikes flag emerging narratives.
Context blockTwo to four sentences with the key facts, caveats, and dates.
Sources listUp to five sources, each with stance pill, publisher, weight bar, and an excerpt.

07 · Workspace home09
07 WORKSPACE HOME

Admins get a Slack-style dashboard.

Open the Veritype app in Slack and you land on its Home tab. KPIs render through Datatype, verdict mix, latency, and channel coverage charts are all live. Recent verifications are one click away.

What admins look at first

  • Claim volume: tells you whether the bot is being used at all, and whether usage is growing.
  • True / False split: an early signal of culture. A workspace where most claims land false is making different bets than one where most land true.
  • Latency: P50 should sit under 7 seconds. If it creeps up, the web_search budget is probably too high.
  • Channel coverage: which channels are over-listened (consider scoping the regex) and which are silent (consider inviting the bot).

08 · Verification library10
08 VERIFICATION LIBRARY

Every claim, every citation, searchable.

The Library tab is the long-tail memory of the bot. Every verification, across every channel, with the full source list intact.

Use it for

  • Recurring debates. "We already verified this in March. Here's the card."
  • Onboarding. New hires can browse the team's litigated facts to learn the house view.
  • Audit. If a claim's status changes (a study is retracted, a number is revised), re-verify and the library tracks the delta.

10 · Settings & tuning12
10 SETTINGS

Tune the listener. Trust the verdict.

Veritype is meant to be quiet by default. The Settings tab is where you make it quieter, louder, or more discerning.

Knobs that matter

SettingWhen to adjust
Auto-listenTurn off if the team prefers explicit /verify only. On is the default.
Watch channels regexUse to whitelist channels. Common pattern: general|product|markets.
Confidence thresholdDefault 0.62. Raise to 0.75+ for very quiet channels; lower to 0.45 for high-signal places like #science-club.
Post in threadDefault on. Off is for channels where the team wants verdicts in-line.
Attach PNG cardDefault on. Off only if you have a workspace-wide rule against image attachments.
Researcher modelOpus 4.7 is the default. Sonnet 4.6 is the fallback and is ~30% cheaper.
Web-search budget5 is a good ceiling. 3 for cost-sensitive workspaces, 8+ for very contested claims.

11 · Setup & deploy13
11 SETUP

From cold clone to live workspace.

Prerequisites

  • Node.js 20+
  • An Anthropic API key with access to Opus 4.7 and the web_search_20250305 tool.
  • A Slack workspace where you can install custom apps.

Install and configure

# 1. clone and install
git clone https://github.com/timdries/veritype.git
cd veritype
npm install

# 2. configure env
cp .env.example .env
# fill in ANTHROPIC_API_KEY, SLACK_BOT_TOKEN, SLACK_APP_TOKEN, SLACK_SIGNING_SECRET

Create the Slack app

  1. Go to api.slack.com/apps and click Create New App → From a manifest.
  2. Paste docs/slack-app-manifest.yaml.
  3. Enable Socket Mode and generate an app-level token with connections:write.
  4. Install to your workspace.

Run

# start the Slack agent (Socket Mode)
npm run dev:slack

# start the demo dashboard (optional)
npm run dev:web

Type /verify Iceland runs on 100% renewable electricity in any channel the bot is a member of. The verdict card lands in thread within ~6 seconds.

Common gotchas. Puppeteer downloads Chromium on first npm install; if behind a corporate proxy set PUPPETEER_DOWNLOAD_BASE_URL. If the bot is silent, double-check the bot is invited to the channel and that the channel name matches the VERITYPE_WATCH_CHANNELS regex.
